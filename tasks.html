(Tentative) Task Description

In this pilot task, we set two subtasks to achieve and advance the technologies related to actionable knowledge graph presentations that can be used for search engines.

Action Mining Subtask (AM):

Input: Entity Type and Entity
Output: A ranked list of actions

For a given entity type (e.g., place) and instance entities (e.g., poland), participants will be asked to find potential actions that can be taken (e.g., traveling to poland, buying a house in poland, finding weather in poland). Participants are allowed to use any external resources (e.g., Action section in schema.org) to return a ranked list of potential actions. This subtask can be seen as an open information extraction task, and allows us to accumulate a comprehensive set of actions that are related to a given entity type. The returned actions will be assessed by crowdsourcing to score 1 to 5. Then, nDCG will be used to evaluate submitted runs.


Actionable Knowledge Graph Generation Subtask (AKGG):

Input: Entity Type, Entity, and Action
Ouput: A ranked list of attributes

For a given entity, entity type, and action (e.g., Kyoto, place, visit a temple), participants will be asked to rank entity properties based on their relevance to the query. The query (input) will be ambiguous as in realistic search queries, and thus, researchers need to return the top N best entity properties to create an actionable knowledge graph. Actions in the testing queries will be taken from the outcomes of the Action Mining (AM) Subtask. Properties to be returned will be those that are defined in the knowledge base. Effectiveness of returned properties will be judged by a crowdsourcing using a variant of simulated work task situation method. Similar to AM subtask, a score 1-5 will be given to each of properties and nDCG will be used to evaluate submitted runs.
